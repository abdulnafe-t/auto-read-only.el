* auto-read-only.el
Automatically activate `read-only-mode' in file-viewing buffers, if those files’ names
match one of the regexps in `auto-read-only-file-regexps'. For example, this can be
used to protect library code provided by third parties.

** Setup
Put this into your =.emacs= file (=init.el=):

#+begin_src emacs-lisp
  (require 'auto-read-only)
  (auto-read-only-mode 1)
#+end_src

Or, with `use-package':
#+begin_src emacs-lisp
  (use-package auto-read-only
    :init
    (auto-read-only-mode 1))
#+end_src

** Customize
#+begin_src elisp
  ;; Automatically activate `read-only-mode' in /vendor/ directories
  (add-to-list 'auto-read-only-file-regexps "/vendor/")
#+end_src

** Alternatives
[[https://github.com/rolandwalker/hardhat][hardhat]] is a package similar to =auto-read-only.el=.
It has large default settings, more flexible and has compatibility with a wide range of Emacs versions.

=auto-read-only= works with a simple mechanism based on emacs’s built-in hook
=window-buffer-change-functions=. It provides an otherwise unobtrusive global minor mode,
and does not interfere with non-interactive Emacs Lisp code (ex. =package-install=).
